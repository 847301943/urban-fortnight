# 回测与数据审计说明

## 本次修改

1. 将原来的“买入判断”拆为两套互不混合的结论：
   - 长期研究结论；
   - 短线技术状态。
2. 短线模块不再使用“60日涨幅代理均线”，而是为高流动性候选获取真实前复权日K，计算均线、RSI、MACD、ATR、突破、回踩和量价。
3. 增加市场环境、行业广度、流动性、过热与保护位判断。
4. 增加历史回测脚本和数据覆盖审计脚本；每次 GitHub Actions 更新后自动生成网页报告。
5. PE、PB、股息率、自由现金流均保留数据口径；缺失不自动加分。
6. 新增应收账款增速相对营收、存货增速相对营收、总股本稀释的批量计算，接口字段缺失时保持空值。

## 本地工程测试

由于当前执行环境不能访问 AKShare 外部接口，本地不能伪造一份“真实A股回测结果”。本次已完成：

- Python脚本编译检查；
- 前端JavaScript语法检查；
- 日K标准化、指标计算、评分和回测流程的确定性合成数据测试；
- 趋势、下跌、突破和震荡四种场景压力测试。

合成测试暴露出初版“趋势跟踪”会在震荡环境频繁开仓的问题，因此已修改为：

- “趋势跟踪”只用于观察，不作为回测入场；
- 回测只交易“突破确认”和“回踩关注”；
- 突破要求真实当日/近期放量、MA20斜率向上、60日收益为正、RSI不过热；
- 回踩要求MA60斜率向上、60日收益为正、缩量且RSI处于正常区间；
- 次日高开超过3.5%不追；
- 市场环境过弱不入场；
- 保护位风险过大或过小会跳过或校正。

最新合成压力测试结果：

- 下跌场景：识别为“弱势规避”，没有开仓；
- 震荡场景：识别为“趋势跟踪/等待”，没有开仓；
- 突破场景：产生1笔触发交易，测试样本净收益约5.98%；
- 持续上升但已过热场景：识别为“过热勿追”，没有追涨。

合成结果仅验证规则行为是否符合设计，不代表真实市场收益。

## 真实A股回测如何生成

上传后运行 GitHub Actions，流程会：

1. 获取全A股行情和财务数据；
2. 选出约320只高流动性、分行业候选；
3. 获取约1250个自然日的前复权日K；
4. 计算最新短线状态；
5. 对最多180只候选执行历史回测；
6. 生成 `data/backtest.json` 和 `data/data_audit.json`；
7. 网页“策略回测与数据审计”区自动展示交易数、胜率、平均收益、盈亏因子和后30%验证结果。

## 仍需关注的缺失

- 事件风险覆盖率目前为0：公告、减持、解禁、停牌与财报窗口必须人工核查；
- 历史回测使用当前存量候选，存在幸存者偏差；
- 部分PE是当前价格除最近完整年度EPS的静态口径；
- 大部分普通企业自由现金流仍是代理口径，只有优先候选补精确资本开支；
- 金融专项字段受公开接口返回范围限制；
- 盈利预测调整仍未自动覆盖。

因此短线模块的定位是“生成盯盘候选与失效条件”，而不是自动给出无条件买入指令。
