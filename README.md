# 大A长期价值与短线信号助手

这是一个无需数据 Token、无需自建服务器的静态选股网页。GitHub Actions 在后台生成：

- `data/stocks.json`：全A股长期基本面、金融行业指标和短线候选最新信号；
- `data/backtest.json`：短线规则的历史工程回测；
- `data/data_audit.json`：字段覆盖率、口径缺失和改进建议。

## 部署

1. 把压缩包内文件上传到 GitHub 仓库根目录，并保留 `.github/workflows/`。
2. `Settings → Pages → Build and deployment → Source` 选择 **GitHub Actions**。
3. `Actions → 更新并发布大A智能选股网页 → Run workflow`。
4. 等待全部步骤变绿，再打开 Pages 网站。

## 两套模型完全分开

### 长期价值视角

- 普通企业：ROE、ROIC、现金流质量、自由现金流、成长、估值和中期趋势；
- 银行：PB、ROE、资产质量、资本充足与股息；
- 保险：PB/P·EV、盈利、偿付能力和业务质量；
- 券商：PB、跨周期盈利、净资本和风控指标。

长期结论是“是否值得进一步研究和分批跟踪”，不是立即交易指令。

### 短线信号视角

后台从全A股中选出约320只高流动性、分行业候选，获取前复权日K并计算：

- MA5/10/20/60/120/250；
- RSI14、MACD、ATR、20日波动率；
- 20/60日突破、MA20回踩、量能和成交额结构；
- 上证指数市场环境、候选池行业广度；
- 技术保护位和参考2R目标。

短线状态包括“突破确认、回踩关注、趋势跟踪、等待信号、过热勿追、弱势规避、流动性不足、数据未覆盖”。只有“突破确认”和“回踩关注”被作为回测入场触发；“趋势跟踪”仅表示趋势存在，不代表新买点。

## 回测规则

- 信号日之后的下一交易日开盘入场，避免收盘后信号使用未来价格；
- 最多持有10个交易日，或触发ATR/MA20保护位；
- 往返成本默认18个基点；
- 同一股票上一笔交易退出后才允许再次开仓；
- 结果拆分为前70%样本和后30%时间验证段。

回测仍有当前存量股票的幸存者偏差、候选选择偏差，未完整模拟历史ST、退市、停牌、涨跌停无法成交、真实滑点和公告事件，所以只用于检查规则方向与工程正确性，不用于承诺收益。

## 当前明确未覆盖

- 公告、减持、解禁、停复牌和业绩披露窗口；
- 盘中盘口、委托队列和真实成交滑点；
- 全市场盈利预测调整；
- 部分公司的应收、存货、股本稀释和金融专项指标；
- 未进入约320只日K候选池的股票短线信号。

网页会把缺失显示为“未覆盖/待核验”，不会把缺失数据当成利好。
