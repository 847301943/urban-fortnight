# 大A长期价值与隔日短线助手

本版以**收盘后隔日计划**为主，长期价值模型只用于财务排雷和研究辅助。系统无需数据 Token，也无需自建服务器；GitHub Actions 每个交易日收盘后重新生成：

- `data/stocks.json`：全A基础数据、长期模型和当日隔日池；
- `data/backtest.json`：隔日规则历史工程回测；
- `data/data_audit.json`：日K刷新率、字段覆盖率和改进建议。

## 隔日模型的真实口径

当前架构在收盘后运行，因此不会假设能够用收盘后才知道的数据在信号日尾盘成交。规则是：

1. T日收盘后生成计划；
2. T+1日开盘相对T日收盘价在 **-4%至+2.5%** 内，才允许进入模拟；
3. 高开超过2.5%不追，低开超过4%取消；
4. A股实行T+1，T+1买入后不能当日卖出；
5. 最早在T+2触发保护位、1.5R目标或收盘退出。

因此这是“收盘后选股、次日介入、短持1个隔夜”的现实模型，不是理想化的“信号日收盘买、次日卖”。

## 当日隔日池，不做轮换

每次运行都从最新全市场快照重新建立当日池，并对入选股票在本次运行中刷新日K。默认门槛：

- 非ST、非退市风险；
- 有有效价格；
- 股价2至200元；
- 当日成交额至少1.5亿元；
- 换手率不过度异常；
- 最多刷新1200只，按成交额、活跃度与当前强度排序。

未进入隔日池通常是流动性或交易性不适合本策略，并不是“数据抓取失败”。网页默认只显示本日已经刷新日K的隔日池。

## 隔日信号

- **隔日突破备选**：收盘突破前20日高点、收盘位置较强、量能确认、不过热；
- **隔日回踩备选**：MA10/20/60多头结构，回踩MA20附近、缩量且收盘企稳；
- **收盘强势观察**：收盘位于日内高位、量价尚可，但不是标准突破或回踩；
- **等待隔日触发**：趋势尚可，缺少明确触发；
- **过热不追、弱势规避、流动性不足**：本日不进入计划。

关键指标包括真实MA10/20/60、3日和5日动量、RSI、MACD、ATR、收盘位置、上影线、当日量相对20日均量、20日平均成交额、市场和行业环境。

## 回测规则

- 信号只使用T日及以前数据；
- T+1开盘介入，跳空范围-4%至+2.5%；
- T+1买入当日不允许止损或止盈，符合A股T+1；
- 默认T+2收盘退出，或T+2触发保护位/1.5R目标；
- 往返交易成本默认20个基点；
- 同一股票不重叠持仓；
- 分前70%样本和后30%验证段展示。

回测仍有幸存者偏差、当前候选选择偏差，未完整模拟历史ST、退市、停牌、涨跌停无法成交、公告事件和真实滑点，只用于检查规则是否具有统计价值。

## 部署

1. 将压缩包中的全部文件覆盖到GitHub仓库对应位置；
2. `Settings → Pages → Source` 选择 **GitHub Actions**；
3. `Actions → 更新并发布大A智能选股网页 → Run workflow`；
4. 首次运行可能耗时较长，因为最多需要刷新1200只股票的历史日K；
5. 全部步骤变绿后打开Pages网站。

不要只替换 `index.html`，后台脚本和工作流必须一起更新。
